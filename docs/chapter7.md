# 7. 統合のテスト実行

この章では、Oracle Integration のデザイナ上から統合をテスト実行し、動作確認を行います。

この方法では、統合をアクティブ化せずに簡単に動作確認ができます。
Integration Canvas からのテスト実行は、設計と検証を素早く繰り返すための重要な機能です。

## 7.1 テスト実行の概要

Oracle Integration では、Integration Canvas 上のテスト機能を使用して統合をその場で実行できます。

以下の流れで動作確認を行います。

1. **「テスト」**ボタンをクリック
2. リクエスト・パラメータの入力と実行
3. 実行結果を確認

## 7.2 「テスト」ボタンをクリック

統合が保存されていない場合は、**「」** ボタンをクリックして統合を保存します。
Integration Canvas のツール・バーにある **「テスト」** ボタンをクリックします。

![スクリーンショット: 「テスト」ボタン](./images/test_button.png)

画面右側に表示された **「統合のテスト」** パネルに統合のアクティブ化（実行可能な状態に変更）の進捗状況が表示されます。

## 7.3 リクエスト・パラメータの入力

統合がアクティブ化されると、 **「統合のテスト」** パネルで、リクエスト・パラメータの入力が可能になります。
「本文」タブをクリックし、次の入力例のように Oracle Integration に送信される JSON ペイロードを指定します。

```json
{
  "product_id": 30
}
```

![スクリーンショット: 「統合のテスト」画面でパラメータを入力](./images/test_input.png)

入力後、 ***「実行」*** ボタンをクリックします。

## 7.4 実行結果の確認

統合が実行されると、画面の右側にアクティビティ・ストリームが表示されます。

![スクリーンショット: アクティビティ・ストリーム](./images/activity_stream.png)

すべてのステップが正常に実行されると、Integration Canvas の各アクションが緑色で表示されます。

アクティビティ・ストリームは統合の実行状況やエラー確認に使用する重要な機能です。
また、各ステップをクリックすることで、処理の詳細やメッセージ内容を確認できます。

アクティビティ・ストリームの確認したら、 **「テステの終了」** ボタンをクリックします。

## 7.5 CSVファイルの確認（SFTP）

本チュートリアルで作成した統合の実行が成功すると、CSVファイルが Oracle Integration のファイル・サーバーにアップロードされます。

Oracle Integration のファイル・サーバーは、SFTP に準拠したサーバーとして提供されており、外部の SFTP クライアントからアクセスできます。
本チュートリアルでは、OCI Cloud Shell から SFTP コマンドを使用してファイルがアップロードされたことを確認します。

### 7.5.1 SFTP 接続情報の確認

Oracle Integration の内蔵ファイル・サーバー（SFTP サーバー）の接続情報は、サービス・コンソールから確認できます。

ナビゲーションメニューから以下を選択します。

- 「設定」→「ファイル・サーバー」→「設定」

![スクリーンショット: ファイル・サーバー設定画面](./images/file_server_settings.png)

この画面で、以下の接続情報を確認します。

- ホスト名
- ポート番号

### 7.5.2 Cloud Shell から SFTP 接続

OCI Cloud Shell を開き、以下のコマンドを実行します。
<ユーザー名> はOracle Integration のログイン時に使用したユーザー名と置き換えてください。

```sh
sftp -p <ポート番号> <ユーザー名>@<ホスト名>
```

Cloud Shell から SFTP ファイル・サーバーへの初回接続時には、以下のようなメッセージが表示されます。

```sh
Are you sure you want to continue connecting (yes/no)?
```

`yes` を入力して Enter キーを押します。

続いて、パスワード (Oracle Integration のログイン時に使用したパスワード) を入力します。

### 7.5.3 ファイル一覧の確認

SFTP ファイル・サーバーに接続されたら、以下のコマンドでファイル一覧を表示します。

```sh
ls
```

以下のように、CSVファイルが表示されていれば成功です。

```sh
output.csv
```

### 7.5.4 ファイルのダウンロード（任意）

ファイルの内容を確認する場合は、以下のコマンドを実行してファイルをダウンロードします。

```sh
get output.csv
```

ダウンロード後、以下のコマンドで内容を確認できます。

```sh
cat output.csv
```

### 7.5.5 接続の終了

SFTP 接続を終了するには、以下のコマンドを実行します。

```sh
exit
```

## 7.6 トラブル・シューティング

### ケース1：SQLエラー

- 原因: SQLの記述ミス
- 対処方法: ATP Adapter の設定を確認

### ケース2：FTP送信エラー

- 原因: 接続情報の誤り
- 対処方法: FTP Adapter の接続設定を確認

### ケース3：パラメータ未設定

- 原因: 必須パラメータ未入力
- 対処方法: 入力値を確認

## 7.7 この章のまとめ

この章では、Integration Canvas 上のテスト機能を使用して統合の動作確認を行いました。

- Integration Canvas からワン・クリックで統合を起動
- アクティビティ・ストリームで処理の流れを確認
- SFTP サーバーにCSVファイルが出力されたことを確認

次の章では、明示的に統合をアクティブ化し、外部から実行してみます。
